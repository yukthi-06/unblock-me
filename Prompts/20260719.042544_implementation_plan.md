# Auto-animate Target Block to Exit

When the path between the target (red) block and the exit is completely clear, the game should automatically animate the block sliding to the exit and trigger the win condition.

## Proposed Changes

### `GameEngine.java`
- Add a new method `isPathClearForTarget()`:
  - Finds the target block.
  - Checks every other block to see if any of them occupy the cells to the right of the target block (from `target.x + target.length` to `GRID_SIZE - 1` on the same `y` row).
  - Returns `true` if no blocks are blocking the path.
- Change `checkWin()` to only trigger if the block is actually at the edge.

### `GameView.java`
- Introduce state variables for animation:
  - `private boolean isAnimatingWin = false;`
  - `private float animatedTargetX = -1f;`
- In `onTouchEvent()`, after a successful move (in `ACTION_MOVE` or `ACTION_UP`), if `engine.isPathClearForTarget()` is true:
  - Disable further touch events (`isAnimatingWin = true`).
  - Use a `ValueAnimator.ofFloat()` to animate the target block's X position from its current grid position to `6 - target.length` (the right edge).
  - Add an `AnimatorUpdateListener` to update `animatedTargetX` and call `invalidate()`.
  - Add an `AnimatorListener` to handle completion:
    - Update the engine's target block position.
    - Manually trigger `engine.checkWin()` or `listener.onWin()`.
- Update `onDraw()`:
  - When drawing the target block, if `isAnimatingWin` is true, use `animatedTargetX * cellSize` instead of `b.x * cellSize`.
