# use assets from 'assets\tiles\wood\*' to draw the tiles

This plan outlines how to integrate the wood tile assets into the game's drawing logic.

## Proposed Changes

### Resource Management
Copy the files from the project root `assets\tiles\wood\` into the Android module's `app\src\main\res\drawable\` directory. They will be renamed to comply with Android resource naming conventions (lowercase, underscores):
- `red_block_1.png` -> `target_block.png`
- `wood_block_183x90_2.png` -> `wood_block_h2.png`
- `wood_block_276x90_0.png` -> `wood_block_h3.png`
- `wood_block_90x183_3.png` -> `wood_block_v2.png`
- `wood_block_90x276_4.png` -> `wood_block_v3.png`

### `GameView` Updates

#### [MODIFY] [GameView.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Games/00-My-Own/Unblock_Me_APK_GITHUB/app/src/main/java/com/vypeensoft/unblockme/GameView.java)
- Decode the new drawable resources into `Bitmap` objects during initialization (or lazy-loaded) scaled to the `cellSize`.
- In `onDraw()`, instead of drawing colored rounded rectangles for blocks, we will draw the appropriate `Bitmap` based on the block's properties:
  - If `b.isTarget` is true, draw the target block asset.
  - If `b.isHorizontal`:
    - Length 2: draw `wood_block_h2`
    - Length 3: draw `wood_block_h3`
  - If `!b.isHorizontal`:
    - Length 2: draw `wood_block_v2`
    - Length 3: draw `wood_block_v3`
