# Level Packs Implementation Plan

Based on your request, each folder under `/sdcard/Vypeensoft/Unblock_Me/game/levels/` will be treated as a "Level Pack" (e.g., easy, hard, 6x6). I will introduce a new screen to allow players to select a pack before selecting a level.

## Proposed Changes

### 1. LevelManager
- Add a new method `public static List<String> getPacks()` to read the directory `/sdcard/Vypeensoft/Unblock_Me/game/levels/` and return a list of subdirectory names (the level packs).

### 2. New Activity: PackSelectActivity
- Create `PackSelectActivity.java` to display a list of all available packs (folders found on the SD card).
- Create its layout `activity_pack_select.xml`.
- When a pack is tapped, it will start `LevelSelectActivity`, passing the selected pack name as an Intent extra (`PACK_NAME`).

### 3. MainActivity
- Update the "Select Level" button to navigate to `PackSelectActivity` instead of directly to `LevelSelectActivity`.
- For the "Play" button (which resumes the game), we will either load the most recently played pack or default to the first available pack.

### 4. LevelSelectActivity
- Read the `PACK_NAME` from the Intent.
- Fetch levels using `LevelManager.getLevels(packName)`.
- When a level is clicked, start `GameActivity`, passing **both** `PACK_NAME` and `LEVEL_INDEX`.

### 5. GameActivity
- Read `PACK_NAME` and `LEVEL_INDEX` from the Intent.
- Load levels using `LevelManager.getLevels(packName)`.
- Update the `GAME_PROGRESS` SharedPreferences to track progress per-pack (e.g., using `LAST_COMPLETED_` + `packName`).

### 6. AndroidManifest.xml
- Register `PackSelectActivity`.

## Verification
- Run the application.
- Ensure the app lists "easy", "hard", "6x6", etc. as packs if those folders exist.
- Selecting a pack shows the correct levels.
- Playing levels correctly loads and saves progress per-pack.
